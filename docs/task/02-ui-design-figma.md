# 02: UIデザイン制作（Figma）

## 概要

Figma MCP を使って、Lunchy の全画面のUIデザインを作成する。実装前にデザインを固めることで、手戻りを減らす。

## 進め方

**Figma での制作に入る前に、必ず `/grill-me` でUIデザインの詳細を徹底的に詰めること。**

### Grill-Me で決めるべき項目

以下の各画面について、1画面ずつ grill-me セッションで深掘りする。

- カラーパレット・タイポグラフィ・全体のトーン&マナー
- 各画面のレイアウト構成（何をどこに配置するか）
- コンポーネントの具体的な見た目（ボタン形状、カードデザイン、アイコン等）
- インタラクション（タップ時の挙動、遷移アニメーション、ローディング表示）
- エッジケースの表示（0人参加時、エラー時、長い店名の場合等）
- 選択肢の具体的な文言（アレルギー項目、予算レンジ、場所の選択肢）
- 空腹度の入力UI（スライダー / ボタン / 星 etc.）
- 待機画面の演出（何を見せて退屈させないか）
- 結果発表の演出（盛り上がり要素）

### フロー

```
/grill-me UIデザイン詳細
    → docs/ui-design.md に決定事項を記録
    → Figma で1画面ずつ制作
    → 1画面制作するごとにユーザーにレビュー依頼
    → フィードバック反映 → 承認後に次の画面へ
```

### レビュープロセス

**1画面ごとに必ずユーザーレビューを挟む。承認されるまで次の画面に進まない。**

1. 1画面を Figma で制作（スマホ版 + PC版）
2. ユーザーにレビュー依頼（Figma のリンクまたはスクリーンショットを提示）
3. フィードバックがあれば修正
4. ユーザーが承認
5. 次の画面へ進む

レビュー順序:

1. トップページ
2. セッション作成フォーム
3. URL共有画面
4. メンバー参加画面
5. 好み入力フォーム（5ステップ）
6. 入力完了・待機画面
7. 幹事ダッシュボード
8. 投票画面
9. 決選投票画面
10. 結果表示画面

## 作成する画面

### 1. トップページ

- サービスロゴ・タイトル
- 「ランチセッションを作る」ボタン

### 2. セッション作成フォーム

- 表示名入力
- オフィスの場所入力
- 作成ボタン

### 3. URL共有画面

- 発行されたURL表示
- コピーボタン

### 4. メンバー参加画面

- 表示名入力
- 参加ボタン

### 5. 好み入力フォーム（5ステップ）

- Step 1: アレルギー選択
- Step 2: 肉 / 魚 / それ以外
- Step 3: 空腹度スライダー（0-10）
- Step 4: 食べたい場所（スキップ可）
- Step 5: 予算選択
- 各ステップの進捗インジケーター

### 6. 入力完了・待機画面

- 「回答済み！」メッセージ
- 待機アニメーション

### 7. 幹事ダッシュボード

- N/M 進捗表示
- 締め切りボタン
- 投票フェーズの進捗表示

### 8. 投票画面

- 候補3件のカード表示（店名・ジャンル・徒歩時間・評価・価格帯・写真）
- 選択UI
- 投票ボタン

### 9. 決選投票画面

- 上位2件のカード表示
- 選択UI

### 10. 結果表示画面

- 決定した店の情報（大きく表示）
- 投票内訳
- Google Maps リンク

## デザイン方針

- スマホ（375px）とPC（1280px）の2サイズで作成
- 明るく親しみやすいカラートーン（ランチ＝楽しい）
- 操作ステップが明確に分かるUI
- タップしやすい大きめのボタン
