# ワークフロー

## 実装サイクル

すべての実装は以下のループを回す。スキップしない。

1. **探索**: コードベース調査、関連ドキュメント確認（Explore エージェント）
2. **計画**: 実装方針をユーザーに提示し承認を得る
3. **実装**: 下記のバックエンド/フロントエンド別ルールに従う
4. **検証**: テスト実行、動作確認、ドキュメントとの整合性チェック

### バックエンド: TDD

```
探索 → 計画 → [テスト記述(Red) → 最小実装(Green) → リファクタ] → 検証
```

詳細は [テスト方針](testing.md) を参照。

### フロントエンド: コンポーネント駆動 + ビジュアルチェック

```
探索 → 計画 → [コンポーネント単体 → 組み合わせ → ページ → Playwright確認 → 崩れ修正ループ] → 検証
```

詳細は [テスト方針](testing.md) を参照。

## 実装開始前の必須確認

- `docs/task/XX-*.md` を読む
- `docs/lunchy-requirements.md` の該当 Decision を確認する

## 作業委託

- Opus: 指揮・判断・ユーザー対話
- Sonnet (general-purpose): コード実装・修正・テスト
- Opus Explore: コードベース調査・分析

## ドキュメントとの乖離時

実装を止めて AskUserQuestion でユーザーに確認する。

## タスク完了時

1. タスクドキュメントのチェックリストが全完了していることを確認
2. テスト通過を確認（API の場合）
3. `docs/task/XX-*.md` の status を `completed` に更新、完了日を記録
