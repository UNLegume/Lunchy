# 08: メンバー参加画面

## 概要

共有URLにアクセスしたメンバーが表示名を入力して参加する画面。

## 画面仕様

### 参加画面 (`/session/[id]`)

- セッション情報の表示（幹事名など）
- 表示名入力フィールド
- 「参加する」ボタン
- 参加後、好み入力画面へ遷移

### エラーケース

- セッションが存在しない → エラーメッセージ表示
- セッションが既に締め切り済み → 「このセッションは締め切られました」表示

## やること

- [ ] 参加画面 UI 作成
- [ ] `POST /api/sessions/[id]/join` の実装
- [ ] メンバーID の発行とブラウザへの保存（cookie or localStorage）
- [ ] セッション状態のバリデーション
- [ ] 参加後の好み入力画面への遷移

## 完了条件

- URLにアクセスして表示名入力→参加ができる
- 参加したメンバーがセッションデータに追加される
- 存在しないセッション/締め切り済みの場合にエラー表示される
