# 09: 好み入力フォーム

## 概要

メンバーが選択式で好みを入力するステップフォーム。

## 画面仕様

### ステップ1: アレルギー

- 複数選択可
- 選択肢（候補）: 卵、乳製品、小麦、そば、落花生、えび、かに、なし
- ※ 具体的な項目リストはOpen Question（実装時に確定）

### ステップ2: カテゴリ

- 単一選択
- 「肉」「魚」「それ以外」

### ステップ3: お腹空いている度

- スライダー or ボタン選択
- 10（めちゃくちゃ空いてる）〜 0（もう食べたくない）

### ステップ4: 食べたい場所

- 単一選択 + スキップ可
- 選択肢（候補）: 駅前、オフィス近く、商店街、スキップ
- ※ 具体的な項目はOpen Question

### ステップ5: 予算

- 単一選択
- 選択肢（候補）: 〜500円、〜800円、〜1,000円、〜1,500円、こだわらない
- ※ 具体的なレンジはOpen Question

### 送信完了

- 「回答済み！みんなの入力を待っています...」の待機画面
- 幹事が締め切るまで待機

## やること

- [ ] ステップフォーム UI 作成（5ステップ）
- [ ] ステップ間のアニメーション/遷移
- [ ] 戻るボタン（前のステップに戻れる）
- [ ] `POST /api/sessions/[id]/preferences` の実装
- [ ] 送信完了後の待機画面
- [ ] 重複送信の防止（同一メンバーの再送信をブロック）

## 完了条件

- 5ステップを順番に進んで回答を送信できる
- 送信データが KV のセッションに保存される
- 送信後に待機画面が表示される
